# AGENTS.md

## Project Overview

MarkEdit is a free, open-source, native Markdown editor for macOS. It uses a **hybrid architecture**: a Swift/AppKit shell hosts a CodeMirror 6-based TypeScript editor inside a WKWebView. The two sides communicate over a message-passing bridge whose Swift bindings are **auto-generated** from TypeScript interfaces via [ts-gyb](https://github.com/microsoft/ts-gyb).

## Repository Layout

```
CoreEditor/          TypeScript editor (CodeMirror 6, Vite, Jest)
MarkEditMac/         macOS application (Swift, AppKit, Xcode)
  Modules/           Feature packages (SPM): AppKitControls, DiffKit, Previewer, etc.
MarkEditCore/        Platform-independent core (Swift SPM): config, shared types
MarkEditKit/         Native <-> Web bridge layer (Swift SPM, auto-generated code)
MarkEditTools/       Dev tooling (SwiftLint plugin)
PreviewExtension/    QuickLook preview extension
FinderExtension/     Finder toolbar integration
```

## Critical: Auto-Generated Code

Files under these paths are **generated by ts-gyb** and must **never be edited by hand**:

- `MarkEditKit/Sources/Bridge/Native/Generated/`
- `MarkEditKit/Sources/Bridge/Web/Generated/`
- `MarkEditCore/Sources/EditorSharedTypes.swift`
- `MarkEditCore/Sources/EditorConfig.swift` (generated from `CoreEditor/src/config.ts`)

To change bridge interfaces, edit the TypeScript source in `CoreEditor/src/bridge/` or `CoreEditor/src/config.ts`, then run `yarn codegen` from `CoreEditor/`.

## Build & Test

```bash
# CoreEditor (TypeScript)
cd CoreEditor && yarn install && yarn build && yarn test

# Swift (via Xcode CLI)
xcodebuild build -project MarkEdit.xcodeproj -scheme MarkEditMac -destination 'platform=macOS'
xcodebuild test  -project MarkEdit.xcodeproj -scheme MarkEditCoreTests -destination 'platform=macOS'
xcodebuild test  -project MarkEdit.xcodeproj -scheme ModulesTests -destination 'platform=macOS'
```

The `yarn build` step runs lint, codegen, and two Vite builds (standard + light theme) in sequence.

## Code Style

### Swift

- Enforced by SwiftLint (`.swiftlint.yml`, 90+ opt-in rules). Runs as an SPM build plugin on every target.
- File header required: `// FileName.swift // TargetName // // Created by author on date`.
- 2-space indentation. Trailing commas mandatory in multiline collections.
- `private` over `fileprivate`. No force unwrapping/casting. Prefer `Self` in static references.
- Strict concurrency enabled (`StrictConcurrency` experimental feature).
- Protocol conformances go in extensions.

### TypeScript

- Enforced by ESLint (`eslint.config.mjs`).
- Single quotes, semicolons required, 2-space indentation.
- No `any`, no non-null assertions (`!`), strict boolean expressions.
- Arrow functions forbidden as class properties ã use methods.
- Accessibility: `no-public` (omit `public`, mark everything else explicitly).
- Prefer `await` over `.then()`.

## Architecture Patterns

### Web-Native Bridge

TypeScript defines interfaces in `CoreEditor/src/bridge/`:
- `native/*.ts` ã calls from Web -> Native (e.g., `NativeModuleCore`)
- `web/*.ts` ã calls from Native -> Web (e.g., `WebModuleCore`)

ts-gyb reads these and generates Swift classes in `MarkEditKit/Sources/Bridge/`.

Manual Swift implementations that handle native module calls live in:
- `MarkEditKit/Sources/Bridge/Native/Modules/EditorModule*.swift`

### Module System

Feature code is organized into small, focused modules in both layers:
- **TypeScript**: `CoreEditor/src/modules/` (completion, search, formatting, etc.)
- **Swift**: `MarkEditMac/Modules/Sources/` (SPM packages)

### Contribution Philosophy

Bug fixes are welcomed as PRs. New features should generally be implemented via forks ã the project intentionally stays minimal. No new language support is accepted.
